# README

## Description

For my class Biol 430 Specialized Topics, 'Reproducibility in Scientific Research', UBCO. This is a Mini Project using public accessible data, to create a mock manuscript from publicity available data and set up a study with reproducible steps.

In this project I have created tables and graphs to compare skull sizes between male and female grizzly bears in BC. This was downloaded from the data "Compulsory Inspection Data 1900 to 2023-24" from the British Columbia database.

This project is composed of a Manuscript, written on RStudio and linked to Github and OpenScience Forum. Sources also linked to a Zoetro.

This project includes the original data in Excel, a Data Directory in Excel, and public README file associated with the original data.

OpenScience Forum linked to Github, including a Data Management Plan, and Pre-Registration in the OSF files.

## Data Collection

The Data Used is "Compulsory Inspection Data 1900 to 2023-24" This data package contains the collected inspection of animals reported by hunters within BC as mandated by BC Wildlife Act, in the form of excel of the raw data and separately the field definitions and supplementary link for more information. Provided with all data, data directory and readme file downloaded 01/23/26.

When running the analysis each data frame should be added to RStudio Environment.

## Data Processing

The data was filtered through RStudio, and all graphs and tables were also created by RStudio.

Under Github files on OSF will be two rmd file titled "mock_manuscript_draf" and "mock-manuscrif_V2" both contain the same script and code. Included will be a pDF of "mock-manuscrifV2" that shows the results of the code.

Under OSF files will be the a rawdata folder containing the original excel data (rawdata_original) from BC Wildlife which was used initially.

To run the analysis use the code in order from top to bottom from either "mock_manuscript_draft" or "mock-manuscrif_V2", replacing the file paths.

Under OSF files will be the Pre-registration and Data Managment plan as pDF's.

-   Users on a different operating system should run renv::restore() to reinstall packages from source.
